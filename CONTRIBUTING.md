# Contribuindo

Obrigada pelo interesse no Fica pro Café! 🤎

Esse é um **projeto pessoal**, mantido por uma única pessoa. Pra manter a coerência da curadoria de perguntas e da direção do produto, **não estou aceitando Pull Requests no momento**.

Mas se você encontrou algo que pode melhorar, sua ajuda é muito bem-vinda nas formas abaixo.

---

## Como mandar feedback

Tem **2 caminhos** dependendo do seu perfil:

### Sou usuária(o) comum

Se você está usando o app e quer reportar bug, sugerir feature ou dar qualquer feedback, **use o botão "Feedback" dentro do próprio app** — ele abre um formulário curto e amigável.

O formulário já vem com **dados técnicos pré-preenchidos automaticamente** (navegador, sistema, idioma, tema atual, versão), então você só precisa descrever o que aconteceu. Sem login, sem complicação.

Você pode:
- 🐛 Reportar um bug
- 💡 Sugerir uma melhoria
- ❓ Sugerir perguntas novas
- 💬 Mandar qualquer outro feedback

### Sou desenvolvedora(o) ou já uso GitHub

Se você é dev ou já tem conta no GitHub e prefere reportar tecnicamente, pode **abrir uma issue diretamente no GitHub**:

- [Abrir issue de bug](https://github.com/dmtamiya/fica-pro-cafe/issues/new?labels=bug)
- [Abrir issue de feature](https://github.com/dmtamiya/fica-pro-cafe/issues/new?labels=enhancement)

---

## 🐛 Reportando um bug — checklist

Pra eu conseguir reproduzir e resolver, ajuda muito ter:

- **O que aconteceu?** Descrição do problema em 1-2 frases
- **O que você esperava que acontecesse?**
- **Como reproduzir?** Passos: 1, 2, 3...
- **Onde aconteceu?** App instalado (PWA) ou pelo navegador?
- **Qual navegador/sistema?** (Chrome no Android, Safari no iPhone, etc)
- **Print ou vídeo** (se conseguir, ajuda muito)

Se você usar o formulário no app, vários desses campos já vêm preenchidos. Pelo GitHub Issue, vai precisar preencher manualmente.

---

## 💡 Sugerindo uma feature — checklist

- **O que você gostaria?** Descrição em 1-2 frases
- **Por que isso seria útil?** Cenário concreto onde você usaria
- **Já existe algo parecido em outro app?** (referência ajuda)

Antes de sugerir, dá uma olhada na [`docs/PRODUCT.md`](docs/PRODUCT.md) — tem uma seção "Decisões deliberadas" que lista coisas que **conscientemente decidi não fazer**. Se sua sugestão tá lá, provavelmente a resposta é não. Mas se você tem um argumento novo, manda mesmo assim!

---

## ❓ Sugerindo perguntas novas

Adoraria! Mas **as perguntas são curadas pessoalmente** pra manter um certo tom (instigantes, mas não invasivas; divertidas, mas não bobas).

Manda sua sugestão pelo formulário no app (categoria "Sugerir perguntas") ou via GitHub Issue, descrevendo:

- **A pergunta** (pode mandar várias)
- **Em qual categoria você acha que encaixa?** (Profundas / Divertidas / Grupo)
- **Cenário de uso:** quando você imaginaria essa pergunta sendo perguntada?

Vou ler todas. Se encaixar, entra no banco em alguma release futura, com crédito a você (se quiser) no [`CHANGELOG.md`](CHANGELOG.md).

---

## Encontrou um typo na tradução?

Pra correções de português ou inglês (typos, frases estranhas, traduções ruins), pode mandar pelo formulário ou abrir issue. Esse tipo de coisa é simples de incorporar.

---

## O que **não** vou aceitar

Pra deixar claro:

- **Pull Requests** — preciso de tempo pra revisar PR e nem sempre tenho. Issues e formulário são mais leves.
- **Sugestões pra mudar a stack** — o projeto é vanilla por escolha (sem React, Vue, build tool, TypeScript). Não vou converter pra framework.
- **Sugestões pra adicionar tracking/analytics** — privacidade é princípio do projeto. Veja [`docs/PRODUCT.md`](docs/PRODUCT.md).
- **Sugestões pra monetizar** — projeto é gratuito por escolha.
- **Conteúdo ofensivo, discriminatório ou tóxico** em issues ou no formulário — vou deletar e bloquear.

---

## Quer fazer sua própria versão?

Esse projeto é **MIT License** — você pode fazer um fork, mexer no código, mudar perguntas, criar tema novo, traduzir pra outro idioma. Tudo é permitido.

Se você criar uma versão derivada, **adoraria ver** — manda link pra mim numa issue ou pelo formulário.

A única coisa que peço: mantenha o aviso de copyright original no `LICENSE` (é a única condição da MIT).

Veja [`docs/TECHNICAL.md`](docs/TECHNICAL.md) pra entender a arquitetura antes de mexer.

---

## Code of conduct

Espera-se que todos no espaço do projeto (issues, comentários, formulário) tratem outras pessoas com respeito. Comentários ofensivos, discriminatórios ou tóxicos serão deletados e seus autores bloqueados.

Não tenho regimento elaborado — é só ser gente boa.
